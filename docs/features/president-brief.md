# President Brief

A daily AI-generated paragraph that tells each role exactly what's happening across their roster and what to act on today.

## Who uses it and why

Content & Social teams at major labels. They open the dashboard each morning and need an instant read on roster health without scrolling through tables and charts.

## Correct behavior

- Card appears at the top of the Content & Social dashboard
- Shows a time-of-day greeting ("Good morning, Paul.")
- Displays a single 2-4 sentence paragraph generated by Claude Opus
- Paragraph answers: "What's going on?" and "What should I do?"
- "Wavebound Brief" badge indicates AI-generated content
- "Updated [date]" timestamp shows when the brief was last generated
- "Got it" dismisses the card for the current session
- "Chat about this" navigates to the AI assistant with the brief text prefilled
- Generated daily at 04:00 UTC on the backend

## Edge cases

- **No brief exists yet:** Falls back to the legacy client-side `ContentBriefingCard` with template-generated paragraphs and action items
- **Loading state:** Parent dashboard skeleton handles this (no separate skeleton in the brief card)
- **Brief is stale (>24h):** Still shows — the timestamp makes staleness visible
- **Empty roster:** Dashboard shows "Your roster is empty" before the brief card renders

## Data flow

1. Backend cron (04:00 UTC) runs `generate-president-brief.ts`
2. Script queries `roster_dashboard_metrics`, `content_anomalies`, `catalog_tiktok_performance`, recent `intelligence_briefs` action items
3. Compressed data sent to Claude Opus with a tight prompt
4. Result upserted into `president_briefs` table (unique per label + role + date)
5. Frontend `usePresidentBrief` hook fetches latest row via React Query
6. `PresidentBriefCard` renders the text
