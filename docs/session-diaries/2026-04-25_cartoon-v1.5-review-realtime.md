# 2026-04-25 (later) — Cartoon v1.5: Review integration + Realtime + lifted caps

Follow-up to the v1 cartoon Create-menu shipped earlier today. Implements all
five recommendations from the v1 session diary.

## What changed

- `src/components/content-factory-v2/types.ts` — extended `QueueItem` with
  cartoon-pipeline fields: `cartoonChatJobId`, `cartoonScriptId`,
  `cartoonStage`, `cartoonHook`, `cartoonFinalUrl`, `cartoonVoCallInFlight`.
- `src/components/content-factory-v2/cartoonReconciler.ts` (new) — pure
  helpers for the cartoon pipeline. Exports `reconcileCartoonItem(item)`,
  `scriptStatusToStage`, `loadCartoonRuns`/`saveCartoonRuns`,
  `snapshotFromItem`/`itemFromSnapshot`, and the `CARTOON_STAGE_ORDER`
  constant.
- `src/components/content-factory-v2/CartoonPanel.tsx` — rewritten as a
  wizard-only component. Removed the inline status board, polling, and
  localStorage logic; all of that now lives in ContentFactoryV2 against
  QueueItems. Bumped `MAX_COUNT` from 3 to 10. Removed the
  "disabled while any in flight" gate; replaced with an informational
  pill so users can keep queueing.
- `src/components/content-factory-v2/CreateView.tsx` — `CreateView` now
  takes `onCartoonGenerate` + `cartoonsInFlight` props and forwards them
  to `CartoonPanel`.
- `src/pages/label/ContentFactoryV2.tsx` — added the cartoon orchestration
  block: hydrate from localStorage on `labelId` change, persist on every
  queue change, `handleCartoonGenerate` (pushes placeholders, fires N
  SSE label-chat streams), Realtime subscriptions on `chat_jobs` /
  `cartoon_scripts` / `cartoon_videos` keyed per phase + 15s polling
  fallback + visibility-change catch-up.
- `src/components/content-factory-v2/ReviewView.tsx`:
  - Hides Risk + Source filter rows whose count is 0 (matches existing
    Artist + Output Type behavior).
  - Adds `cartoon` to the `ALL_OUTPUT_TYPES` filter list.
  - Renders `CartoonStageTimeline` inside QueueCard for cartoon items in
    flight.
  - Adds a `CopyLinkButton` to the action row whenever `cartoonFinalUrl`
    is set.
- `src/hooks/useLabelArtists.ts` — extended `LabelArtist` interface with
  the roster-detail columns (`avatar_url`, `tiktok_followers`,
  `spotify_popularity`, `posting_dates_*`, `last_post_date`, `status`,
  `updated_at`). Dropped the SQL `not("artist_handle", "is", null)`
  filter; callers (CreateView, CartoonPanel) project + filter for
  non-null handles client-side. Restated the docstring to call this the
  single canonical source.
- `src/pages/label/LabelRoster.tsx` — replaced the bespoke
  `useEffect`-driven `supabase.from("artist_intelligence").select(...)`
  with `useLabelArtists(isLabel ? labelId : null)`. Loading/error state
  derives from the React Query hook directly.
- `docs/features/cartoon-create.md` — rewritten to describe the v1.5
  architecture.

## Why

Five follow-ups from the v1 session, all asked for at once:

1. Cartoons should land in Review next to fan briefs (the user trusts the
   pipeline now).
2. Realtime > 6s polling for snappier stage flips.
3. Cap of 3 was too tight — bumped to 10 with no in-flight gate.
4. `useLabelArtists` was promising "single canonical source" but
   LabelRoster still had its own fetch path.
5. Empty filter chips in Review (Source, Risk) added visual noise.

## What was tested

- `npx tsc --noEmit` — clean (no output) at every checkpoint along the
  way: after the QueueItem type extension, after the LabelRoster
  migration, after the ContentFactoryV2 orchestration block, after the
  ReviewView timeline + copy-link.

## What to verify in browser

1. **Wizard cap** — count chips show 1–10. Cost preview updates with
   selection. No "in flight" gate; informational pill appears once any
   cartoon is mid-flight.
2. **Generate** — the active tab switches to Review immediately and one
   QueueCard appears per cartoon, each with the 5-stage timeline showing
   Script highlighted.
3. **Realtime advance** — as the backend flips
   `cartoon_scripts.status` (rendering_vo → vo_complete → rendering_images
   → images_complete → rendering_video → complete), the Stage pills
   advance without a full poll waiting for the 15s timer.
4. **Refresh during a run** — closing the tab and reopening
   `/label/content-factory-v2?tab=review` restores all in-flight + done
   cartoon cards (loaded from localStorage), and the reconciler resumes.
5. **Done state** — the QueueCard's thumbnail (rendered through the
   existing `renderedClipUrl` path) opens the 9:16 MP4 viewer modal. The
   new "Copy link" action button puts `cartoonFinalUrl` on the clipboard.
6. **Filters** — Risk and Source filter rows should be hidden when no
   item in the current tab has that risk/source.
7. **Roster page** — `/label/roster` still loads the same data via the
   shared hook. Frequency window + sort options still behave.

## While I was in here

1. **Active jobs strip** in ContentFactoryV2 only aggregates fan_brief
   placeholders. Cartoon items don't appear in it (they're each their
   own QueueCard already, so the strip would be redundant). Worth
   revisiting if the strip becomes the primary "what's running" surface.
2. **`cartoon_scripts` and `cartoon_videos` are not in the generated
   Supabase Database type** — the reconciler casts via
   `.from("table" as never)`. Once the next typegen runs, those casts
   should be removed.
3. **Cartoon-vo retry race** — `cartoonVoCallInFlight` is in-memory only
   to keep refresh-recovery simple; a hard refresh during the cartoon-vo
   POST will (rarely) cause a duplicate call. To eliminate this, query
   `cartoon_scripts.chat_job_id = X` before firing — but that requires
   confirming the column exists.
4. **Artist filter in Review sidebar** — only shows MOCK_ARTISTS, so
   real cartoon (and fan-brief) items are filterable by Output type +
   Source but not by Artist. Same gap that existed for fan_brief; worth
   fixing once the queue uses real label-roster artist IDs.
