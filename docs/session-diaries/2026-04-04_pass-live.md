### What was fixed

**1. `/label/paid-amplification` → 404 (🟡 BROKEN)**

- Root cause: Sidebar navigates to `/label/amplification`, but no route existed at `/label/paid-amplification` for direct URL access
- Fix: Added `<Route path="paid-amplification">` with `<Navigate to="/label/amplification" replace />`

**2. Light mode: artist cards stay dark (🟢 GLITCH)**

- Root cause: RosterCard used hardcoded `style={{ background: "hsl(0 0% 5%)" }}` ignoring the `data-label-theme` CSS variable system
- Fix: Changed to `style={{ background: "var(--surface, hsl(0 0% 5%))" }}` which reads from `--surface` (white in light, dark in dark)

**3. 768px: artist names truncated to single letters (🟢 GLITCH)**

- Root cause: Grid used `md:grid-cols-3` (768px) while sidebar is still visible (collapses at 1024px). 3 columns + sidebar = ~173px per card = names truncated to 1 char
- Fix: Changed breakpoints from `grid-cols-2 md:grid-cols-3 lg:grid-cols-4` to `grid-cols-2 lg:grid-cols-3 xl:grid-cols-4` so 2 columns persist until sidebar collapses

**4. Light mode: hardcoded dark colors across 5 components (🟢 GLITCH)**

- Root cause: Multiple components used hardcoded `#1C1C1E`, `#111111`, `#0a0a0a` backgrounds and `rgba(255,255,255,...)` text colors
- Fix: Replaced with `var(--surface)`, `var(--bg-subtle)`, `var(--bg)`, `var(--border)`, `var(--ink)` etc. with dark fallbacks
- Components fixed: RosterListView, ArtistActivityFeed, UniversalAddArtistModal, MonitoringTrendChart, SoundComparison

**5. Content X-Ray verdict badge overflow (🟢 GLITCH)**

- Root cause: Verdict badge used `whitespace-nowrap` with no max-width — long AI verdicts (e.g. "STOP LONG-FORM VLOGS...") blow out the table column
- Fix: Added `max-w-full overflow-hidden text-ellipsis` to badge, `max-w-[180px]` to cell, native `title` attribute showing full verdict + description on hover
